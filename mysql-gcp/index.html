Creating MySQL instance using Google Cloud Platform

Posted on October 31, 2019

Creating and Managing MySQL Databases in Google Cloud..

In this article we are going to summarize how to create an instance of MySQL database using Google Cloud Platform, connect to the created instance from Googleâ€™s cloud shell to test it and then connect to the same instance using MySQL workbench on our local machine. There are a lot of ways to work within the databases in the cloud, similar to Microsoft Azure or AWS. In this case we are going to focus on basic Google Cloud Platform and will not include advanced configuration options.

  • Go to https://cloud.google.com/ and create a free trial Google Cloud Platform account. This will give 12 month or $300 free credit (whichever comes first)
  • Go to https://cloud.google.com/ and go to your console.
  • Go to SQL the dashboard section and select create an instance.
  • Select Choose Second Generation (Recommended)
  • Create an instance id and root password.
  • Select Create. This will take some time but create the MYSQL instance in Google Cloud Platform.
  • Once it is complete there will be a checkmark next to the instance ID.
  • Click on the created MSQL instance.
  • Note the Server IP address and Connect Using Cloud Shell. This will take some time to whitelist the client IP address.
  • Cloud Shell will prompt you to enter your password. Upon password entry, the instance connection will be complete. We can further execute SQL statement and access to the MYSQL instance via the Cloud Shell.
  • Go to google.com and get the public IP address.
  • Go to the instance details/Connections and add a network and add this IP address under the network name and click save.
  • Go to MYSQL workbench in local machine.

    • We were able to retrieve the coordinates and corresponding world heritage sites. This completes the web scraping and data preparation process for further analysis.

  • Create a new connection.
  • Add Connection Name
  • Update the hostname with the noted Server IP address from step 10.
  • Create username and best practice to create non root password.
  • Keep the default schema.
  • Test Connection will prompt you to enter the password. Enter your instance password.
  • Successfully made the MySQL connection should display. Click Ok.
  • The Google Cloud Platform MYSQL is created and connected to local MySQL workbench.

    • The process is very straightforward and easy to follow. We can further create relational databases and tables, perform queries from our local MySQL workbench with GCP instance. We can also load any of the tables created within the database to our local R or Python and perform analysis.
